What Your Current System Is Missing
A thorough gap analysis comparing your existing setup against enterprise-grade admin dashboards. Severity indicates impact on operations and user experience.
⬤ Critical
No Global Command Palette
No Cmd+K search across users, tournaments, orgs. Power users waste time navigating menus. Every top-tier admin panel (Linear, Vercel, Stripe) has this.
⬤ Critical
No Audit Log / Activity Trail
Zero record of who changed what, when. Critical for multi-admin orgs. No way to investigate incidents or track unauthorized changes.
⬤ Critical
No Real-time Alerts / Notifications
No system alerts for broken tournaments, failed emails, DB warnings, or spike in errors. You only find out when something is already broken.
⬤ Critical
No User Growth Funnel
Analytics shows raw counts but not conversion funnel: signups → active orgs → active tournaments → retained users. No cohort analysis.
⬤ Critical
No Health/Status Dashboard
No system health panel showing DB performance, email queue depth, celery task status, error rates, or uptime. Flying completely blind.
⬤ Critical
No Revenue / Billing Module
If NekoTab ever monetizes, there's no billing panel. No Stripe integration, no plan management, no subscription overview.
⬤ Medium
Analytics Lacks Comparison Views
No period-over-period comparison (this month vs last), no trend forecasting, no custom date range pickers.
⬤ Medium
No Customizable Dashboard Layout
Fixed widget grid. Superusers can't pin, rearrange, or choose which metrics they care about. One size fits nobody.
⬤ Medium
Campaigns Lacks Scheduling & Segmentation
Can only send now or manually. No scheduled sends, no audience segmentation (by org, plan, activity), no A/B testing.
⬤ Medium
No Bulk Operations on Users
User management only has delete. No bulk role assignment, no bulk org assignment, no user notes or tagging.
⬤ Medium
No Dark/Light/Custom Theme Toggle
Interface is fixed. No theme customization, no compact/comfortable density options, no accent color preferences.
⬤ Medium
No Org/Tournament Health Score
No at-a-glance health status for organizations (active? stale? broken slug? no tournaments?). Need to drill in manually.
⬤ Enhancements
No Keyboard Navigation / Shortcuts
No keyboard shortcut system. Power admin users rely on shortcuts. Competitors like Linear are entirely keyboard-navigable.
⬤ Enhancements
No Saved Filters / Views
Filters on user and tournament lists reset on every page load. No way to save "active orgs with broken slugs" as a reusable filter.
⬤ Enhancements
No Inline Editing in Tables
Every change requires opening a separate edit form. Inline edit for fields like org name, user role, or email would massively speed up workflows.

06 —
UI/UX Design Principles
The guiding principles that should govern every screen, component and interaction in the redesign.
1
5-Second Rule
Any critical metric (active users, system health, error state) must be visible within 5 seconds of opening the dashboard — no drilling required.
2
Role-Aware UI
Superusers see full controls. Org admins see org-scoped views. Members see limited read-only panels. The UI adapts, it doesn't block with error pages.
3
Progressive Disclosure
Show summary → click for details → click for raw data. Never dump everything on one page. Accordions, drawers, and drill-downs beat monster forms.
4
Always Contextual Help
Tooltips on every metric, empty states with actions, inline hints for complex forms. Users should never get stuck without knowing the next step.
5
Optimistic UI
Actions should feel instant. Inline edits apply immediately with background sync. Undo toasts appear after destructive actions (delete, send, bulk ops).
6
Data with Insight
Never show a raw number alone. Every metric has a trend arrow, comparison delta, or contextual label. Numbers without context cause bad decisions.
7
Mobile as First Class
Bottom tab navigation on mobile. Touch-friendly tap targets (44px min). Collapsible tables with priority columns. Charts that don't require pinch-zoom.
8
Consistency Over Cleverness
Same action always looks the same. Edit always in a drawer. Delete always in a confirmation modal. Filters always in the same corner. No surprises.

Note on HTMX: For inline editing, filter updates, and partial page refreshes — HTMX paired with Alpine.js is the lowest-friction upgrade path for a Django project. It avoids the full React/Vue rewrite while delivering 80% of the interactivity improvement. Consider this the recommended path before a full JS framework migration.
